\section{Motivation}

The MVP simulator uses a binary action space (\texttt{nudge}, \texttt{idle}).
Issue \#126 extends this to 6 clinically-motivated actions with per-action
burden penalties. Each action carries a \texttt{burden\_penalty} that feeds
into the user fatigue model.

\section{Config Schema}

Actions use a \texttt{list[dict]} format where each dict has \texttt{name}
and \texttt{burden\_penalty}:

\begin{verbatim}
actions:
  - name: motivational_prompt
    burden_penalty: 0.2
  - name: idle
    burden_penalty: 0.0
\end{verbatim}

The \texttt{action\_names} property extracts a flat \texttt{list[str]} from
the dicts. Agents receive plain action names with no schema changes needed.

Standard UCB achieves the highest reward (270.4). Contextual agents do not
outperform standard variants on this config, likely because the 6-action
space is too large for activity-only conditioning to learn effectively
within 450 steps.

\section{Open Questions}

\begin{itemize}
\item The penalty values are hand-tuned placeholders. A grid search over
penalty combinations could identify regions where burden penalty meaningfully
shapes agent behaviour.
\item With 6 actions and only 2 states, the transition matrix has 12 entries.
Real clinical data would inform more realistic transition probabilities.
\end{itemize}
